# SD-001 — Mythos Project Charter

Version: 0.1

Status: Approved

Owner: Mythos Executive Development

Last Updated: July 2026

--------------------------------------------

# 1. Purpose

Mythos is a long-term game development initiative focused on creating a reusable living-world RPG framework capable of supporting multiple standalone games across different settings.

Rather than developing a single game, Mythos establishes a common architecture that allows future titles to share systems, tools, and development workflows while presenting unique worlds, stories, technologies, and cultures.

The framework is intended to maximize reuse, scalability, maintainability, and long-term expansion.

--------------------------------------------

# 2. Franchise Vision

Each game is released as an independent title under the Mythos brand.

Examples include:

- Mythos: Kingdoms
- Mythos: Syndicate
- Mythos: Frontier
- Mythos: Aftermath
- Mythos: Galaxy

Each title shares the same underlying framework while supplying its own:

- World
- Lore
- Technology level
- Combat style
- Assets
- Story
- Setting-specific mechanics

--------------------------------------------

# 3. Mission Statement

Create living worlds where players build their own stories rather than follow predetermined paths.

The player is not the center of the universe.

The world exists before them, continues without them, and remembers their actions.

--------------------------------------------

# 4. Core Design Philosophy

## Living World

The world functions independently of the player.

NPCs work.

Businesses operate.

Kingdoms govern.

Wars continue.

Trade flows.

The player joins an existing world rather than creating one.

## Player Freedom

Players define their own success.

There is no universal victory condition.

Possible goals include:

- Building businesses
- Becoming wealthy
- Exploring the world
- Joining factions
- Running kingdoms
- Mastering professions
- Following stories
- Ignoring stories

Every playstyle is considered valid.

## Shared World Rules

Players and NPCs follow the same world rules whenever practical.

If players may:

- Own businesses
- Learn professions
- Build wealth
- Marry
- Create organizations
- Lead kingdoms

NPCs should generally be capable of the same activities.

## Meaningful Consequences

Actions produce lasting effects.

Success must be earned.

Failure is possible.

Failure creates new gameplay opportunities instead of immediately ending the game.

## Long-Term Growth

Artificial progression limits should be avoided.

Players may eventually master many disciplines.

The primary limitation is time, effort, and opportunity rather than arbitrary restrictions.

--------------------------------------------

# 5. Player Role

Every player begins as an ordinary member of society.

The player is not inherently special.

They become influential through their actions.

Players choose their starting region before beginning the game.

Professions are learned through gameplay rather than selected during character creation.

Combat class and profession are independent systems.

--------------------------------------------

# 6. World Philosophy

The world does not wait for the player.

Major events continue naturally.

Players may participate, ignore, or alter these events.

--------------------------------------------

# 7. NPC Philosophy

Named NPCs are citizens rather than decorations.

Every named NPC should possess:

- Identity
- Purpose
- Occupation
- Schedule
- Relationships
- Traits
- Goals
- Ambitions

NPC simulation should scale according to importance while maintaining the appearance of a living society.

--------------------------------------------

# 8. Economy Philosophy

The economy operates continuously.

Not every transaction must be fully simulated.

The framework favors believable outcomes over exhaustive simulation.

Crafting services and work orders should support custom materials, labor, duration, and supply acquisition.

--------------------------------------------

# 9. Simulation Philosophy

The framework uses hybrid simulation.

Nearby systems are simulated in detail.

Distant systems use abstract simulation while preserving believable outcomes.

Simulation fidelity varies according to relevance.

--------------------------------------------

# 10. Time Philosophy

Time progresses continuously.

Certain activities accelerate time naturally.

The world always progresses alongside the player.

Multiple simulation layers operate at different time scales while sharing one calendar.

--------------------------------------------

# 11. Framework Philosophy

The Mythos Framework is independent of any single setting.

Framework systems should remain abstract.

Setting-specific content belongs in modular content packages.

Framework code should never assume fantasy, modern, western, or science-fiction concepts.

--------------------------------------------

# 12. Studio Development Philosophy

Mythos is developed as though it were a professional game studio project.

Executive decisions define architecture.

Codex performs implementation after approval.

Every major system is documented before implementation.

Architecture takes precedence over convenience.

Systems should favor:

- Modularity
- Reusability
- Data-driven design
- Maintainability
- Testability
- Scalability

--------------------------------------------

# 13. Long-Term Objectives

The Mythos Framework should eventually support:

- Living NPC simulation
- Dynamic economy
- Emergent quests
- Organizations
- Businesses
- Kingdoms
- Construction
- Property ownership
- Relationships
- Reputation
- Politics
- Crime
- Religion
- Transportation
- Trade networks
- Dynamic world events

Not every title must include every capability.

--------------------------------------------

# 14. Definition of Success

A successful Mythos title allows players to tell stories that were never scripted.

Players should remember:

"I started as a fisherman."

"I became a merchant."

"I lost everything."

"I rebuilt."

"I founded a kingdom."

"My rival defeated me."

"I ignored the war."

"I became part of history."

Rather than:

"I completed the main quest."

--------------------------------------------

# Mythos Motto

"The world belongs to everyone—not just the player."
